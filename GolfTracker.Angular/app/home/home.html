Welcome to Golf Tracker (Azure DocumentDB)

This is an AngularJS client that interacts with an ASP.NET WebApi 2 service that uses Windows Azure DocumentDB as the data store. It's a reference application that shows some more "real-world" implementations of using DocumentDB with clients in a flexible way.

Overview

This is a list of some of the technologies used in this application.

  • Windows Azure DocumentDB
  • ASP.NET WebApi 2
  • Angular JS

I built this application as a proof-of-concept to test whether I could use DocumentDB in a certain fashion that is needed for an upcoming project.

Single Collection Concept

Since DocumentDB is a new NoSql offering from Azure, and since I'm new to NoSql I wanted to "test the waters", with some concepts I needed to use with DocumentDB.

The single-collection concept (which I'm coining), means that due to certain pricing policies, I need to initially store all of my data in a single collection. So I needed an easy-to-replicate way of handling this when I wanted to create a new "schema". This application demonstrates this.

You can find out more about this at http://www.nosqlcentral.net.
